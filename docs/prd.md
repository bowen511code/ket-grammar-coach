# KET Grammar Coach — 产品说明（PRD）初稿

## 1. 项目背景

- **场景**：KET 备考中的语法练习，学生做题、可请求提示，教师查看作答情况。KET 语法练习通常以静态题目和标准答案为主，缺少即时提示与反馈；本项目希望通过一个最小可运行的 Demo，展示「学生练习 + 提示 + 反馈 + 教师查看记录」的基础闭环。
- **定位**：一个独立的 AI 语法练习 Demo 项目，用于展示产品设计、前后端联调与最小可运行 MVP 的实现能力；优先可运行、可演示、可部署。

## 2. 目标用户

- **学生**：做语法题、要提示、提交答案并得到对错与反馈。
- **教师**：查看学生作答记录（MVP 不做用户体系，可视为“匿名作答记录”）。
- **本人/评审**：通过 Demo 入口页了解项目，并通过学生端/教师端页面体验功能。

## 3. MVP 范围

- **学生端**：获取一道题、请求提示、提交答案，看到 correct / feedback / expected_answer。
- **教师端**：查看 attempts 列表（题目 id、答案、对错、时间）。
- **展示层**：Demo 入口 `/`，以及学生端 `/demo/student`、教师端 `/demo/teacher`。
- **后端**：提供 `GET /health`、`GET /api/question`、`POST /api/hint`、`POST /api/submit`、`GET /api/attempts`（约定见 [api-spec.md](api-spec.md)）。

## 4. 成功标准

- 面试/演示场景下能流畅展示：Demo 入口页 → `/demo/student`、`/demo/teacher`。
- 本地可运行前端 + 后端（+ 可选 DB）。
- 具备简明文档（PRD、API 约定、架构说明），便于讲解与后续扩展。

## 5. 非目标 / 边界（MVP 不做）

- 用户注册、登录、权限体系。
- 多套题目、复杂题库管理、题目 CRUD 后台。
- 支付、社交、推送等。
