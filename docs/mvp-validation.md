# MVP 验收记录

## 验收背景

KET Grammar Coach 已完成本地最小可行产品（MVP）开发，实现学生端「题目 → 提示 → 提交」与教师端「查看作答记录」的闭环。本文档记录基于既定验收标准（AC1–AC6）的验证结果

## 验收环境

| 项目     | 说明 |
|----------|------|
| 前端     | Next.js，本地地址 http://localhost:3001 |
| 后端     | FastAPI + Uvicorn，本地地址 http://localhost:8001 |
| 数据     | 内存存储，未接数据库 |
| 验证方式 | 人工按 AC 逐条操作并确认 |

## 验收标准与结果

| 编号 | 验收标准 | 验收结果 |
|------|----------|----------|
| **AC1** | Student 页面能显示题目：进入 /demo/student 后能看到题干、语法点、四个选项，无报错。 | ✅ 通过 |
| **AC2** | 获取提示可用：点击「获取提示」后出现提示文案，无报错。 | ✅ 通过 |
| **AC3** | 提交答案并展示结果：选择选项后点击「提交答案」，出现结果区：correct（答对/答错）、feedback、expected_answer 与选择一致。 | ✅ 通过 |
| **AC4** | Teacher 页面能展示 attempts：进入 /demo/teacher 后，有记录时表格展示 id、question_id、answer、correct、created_at；无记录时显示「暂无作答记录，请先在学生端提交答案。」 | ✅ 通过 |
| **AC5** | 同一条提交在 Teacher 可见：在 Student 端提交后，在 Teacher 端点击「刷新记录」，能看到刚提交的那条记录，且 answer、correct、question_id 与提交一致。 | ✅ 通过 |
| **AC6** | 后端健康与 API 可用：浏览器访问 http://localhost:8001/health 返回 `{"status":"ok"}`；/docs 可打开；Student/Teacher 操作无网络错误。 | ✅ 通过 |

**结论**：上述六项验收标准均已在本地环境中验证通过，当前 MVP 闭环满足既定目标。

## 当前已知限制

- **题目**：仅一道固定题（q_001），无动态出题、难度分级。
- **数据**：使用内存数据，不持久化；重启后端后作答记录清空。
- **部署**：线上 Vercel 前端尚未连接线上后端，仅可展示静态/本地联调效果。
- **未实现能力**（留待后续 roadmap）：
  - LangGraph 状态机与练习流程编排
  - 动态出题与错因分类
  - 补救练习生成
  - 指标体系与学情分析
